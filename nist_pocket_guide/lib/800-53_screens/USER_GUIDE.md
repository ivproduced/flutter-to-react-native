## üìã Working with Baselines

### Understanding Impact Levels

**Low Impact Systems:**
- Basic security controls for low-risk environments
- Minimal business impact if compromised
- Example: Public websites, general business applications

**Moderate Impact Systems:**  
- Enhanced security for most organizational systems
- Significant impact if security is compromised
- Example: Internal business systems, employee data

**High Impact Systems:**
- Maximum security controls for critical systems
- Severe impact if confidentiality, integrity, or availability is lost
- Example: National security systems, critical infrastructure

### Using Baseline Views

1. **Access baselines** from the main dashboard or families screen
2. **Browse controls** included in each impact level
3. **Compare baselines** to understand security progression
4. **Export baseline lists** for implementation planning

### Enhanced Control Information

Each control provides:
- **Complete control statement** with implementation guidance
- **Supplemental guidance** explaining the control's purpose
- **Control enhancements** for additional security measures
- **Related controls** showing dependencies and relationships
- **Assessment objectives** for testing control effectiveness

### Control Relationships

Understanding how controls work together:
- **Prerequisites**: Controls that must be implemented first
- **Complementary**: Controls that work together for comprehensive protection
- **Enhanced by**: Optional enhancements that strengthen base controls
- **References**: External standards and frameworks

### Status and Lifecycle

Controls have different statuses:
- **Active**: Current, implementable controls
- **Withdrawn**: Superseded by newer controls  
- **Incorporated**: Merged into other controls
